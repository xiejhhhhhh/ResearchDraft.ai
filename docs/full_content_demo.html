ResearchDraft.ai manuscript outline demo

从 Idea、Data Description 和指定文献集合生成可扩写为发表论文的英文初稿大纲

这个 demo 展示新的输出目标：不是短 proposal，也不是模板化摘要，而是一份能继续扩写成论文的写作骨架。系统先总结参考文献，再围绕研究背景、数据构建、方法设计、预期结果和讨论逻辑生成自然连贯的英文大纲。

1. 文献输入 读取用户选择的 Zotero collection。若本地文献少于 10 篇，再补充外部检索文献。
2. 单篇总结 对每篇文献总结摘要、数据、方法、结果、不足和核心图表线索，并保存为 HTML。
3. 数据与方法 根据用户填写的数据介绍推导预处理、公式、demo_data.py 和 demo_method.py 工作流。
4. 论文大纲 输出 Markdown、LaTeX、BibTeX，并在 submissions.json 只保留元数据和文件路径。

提交示例

Idea: Early Detection of Potato Diseases During the Growing Season Using Multi-Agent Machine Learning

Data Description: Sentinel-2 satellite imagery, ERA5 meteorological reanalysis, ISRIC SoilGrids, UAV crop disease imagery, PlantVillage/Kaggle plant disease images, field surveys, experimental station records, and agricultural monitoring datasets.

Collection: Potato Disease Detection

注意：已有研究方案文档只作为后续评估与对比优化材料，不作为初始 prompt 的已知内容输入。Agent 应先按流程收集文献和数据逻辑，再与参考方案对比。

生成文件结构

data/literature_summaries/<task_id>/existing_01.html Zotero collection 中已有文献的单篇总结。
data/literature_summaries/<task_id>/supplemental_01.html 当本地文献少于 10 篇时，外部检索补充文献的单篇总结。
data/drafts/draft_*.md / .tex / .bib 论文大纲、LaTeX 初稿和 BibTeX 文献库。
data/drafts/demo_data_*.py 数据清洗、归一化、时间聚合和特征融合的可执行雏形。
data/drafts/demo_method_*.py 多智能体预测、融合和风险评分的模型雏形。
data/submissions.json 仅保存提交信息、collection、标题、字数、状态和文件路径。

Literature Synthesis Before Writing

Before drafting the manuscript, the agent writes one compact synthesis for every cited paper. Each synthesis records the paper's research question, data, methods, scientific findings, limitations, and relevance weight. Zotero papers and supplemental search papers are kept distinguishable through their HTML filenames and metadata. The draft then gives stronger narrative space to highly relevant and more authoritative studies, while still making clear where evidence is weak or not directly transferable.

Introduction

The first paragraph introduces the broad problem and core concepts. For the potato disease detection example, it should explain why in-season disease monitoring matters for crop yield, pesticide timing, and precision agriculture, and why remote sensing and machine learning have become central tools for field-scale disease surveillance.

The second paragraph reviews current research status by synthesizing the selected references. It should connect image-based plant disease recognition, UAV and satellite monitoring, environmental drivers of disease risk, and machine-learning classification into a coherent research landscape, with author-year citations such as (Author, Year).

The third paragraph identifies the specific gaps that justify the new study. These gaps should not be generic. They should explain, for example, why many models remain single-source, image-only, spatially local, or weakly validated across growth stages, and how these weaknesses limit early detection during the growing season.

The fourth paragraph states the purpose and value of the study. It should describe what the research does, how multi-agent learning organizes heterogeneous data sources, and why the resulting framework could improve early warning, field management, and scientific understanding of crop disease dynamics.

Data

The first data paragraph introduces all data sources and their roles. In the test case, Sentinel-2 provides field-scale optical time series, ERA5 supplies meteorological context, SoilGrids characterizes soil background, UAV imagery supports fine-grained visual disease evidence, PlantVillage/Kaggle images support controlled visual symptom learning, and field surveys or station records provide ground-truth labels.

The second paragraph explains data construction and preprocessing. It should cover cloud filtering, spatial alignment, temporal aggregation, image normalization, label harmonization, and quality control. A typical vegetation-index or feature-standardization formula can be written as \(z=(x-\mu)/(\sigma+\epsilon)\), while time-window aggregation can be expressed as \(\bar{x}_{t}=\frac{1}{k}\sum_{i=t-k+1}^{t}x_i\).

The third paragraph explains how processed data become model-ready inputs. It should describe multimodal feature fusion, uncertainty handling, class imbalance, train-validation splits by field or season, and the role of `demo_data.py` when no existing GitHub implementation is available.

Methods

The first methods paragraph explains the model architecture. For this example, each data source can be represented by a specialized agent: a satellite time-series agent, a meteorological-risk agent, a soil-context agent, a UAV/image agent, and a label-quality agent. Each agent produces disease evidence that is later fused.

The second paragraph formalizes the fusion logic. A concise expression such as \(p(y|X)=\sum_{m=1}^{M}\alpha_m p_m(y|X_m)\) can show how agent outputs are combined, where \(\alpha_m\) reflects source reliability, temporal relevance, or validation performance.

The third paragraph describes training and evaluation. It should cover loss functions, temporal validation, disease-risk scoring, accuracy metrics, calibration, and ablation analysis. If no paper-specific GitHub code exists, `demo_method.py` provides an auditable scaffold before a full model is implemented.

Results

The first result paragraph should answer the gap that many disease-detection studies are image-only or single-source. It should explain how the proposed data pipeline creates an AI-ready multi-source representation for early-season disease risk, without inserting literature citations.

The second result paragraph should answer the gap that current models often lack temporal and environmental reasoning. It should describe whether multi-agent fusion improves early detection before visible symptoms become spatially extensive.

The third result paragraph should highlight the innovation that the method is not only a classifier but also a decision-support framework. It should connect risk maps, uncertainty, and targeted field scouting into a clear expected scientific contribution.

Figure and Table Plan

The figure plan is now a standalone section after Results. It should follow the manuscript narrative and show how each figure or table resolves a limitation in existing research or demonstrates the new method's innovation.

Discussion

The first discussion paragraph expands the main expected finding: early disease risk is not only an image-recognition problem but also a temporal and environmental inference problem. This gives the study a stronger scientific argument than simply applying a classifier to plant images.

The second paragraph discusses data complementarity. It should explain how satellite, weather, soil, UAV, and field-label information compensate for one another, while also acknowledging scale mismatch, cloud contamination, label uncertainty, and uneven sampling.

The third paragraph discusses model interpretability and deployment. It should address whether agent-specific outputs can help agronomists understand why a field is classified as high risk and whether the method could support seasonal monitoring rather than one-time diagnosis.

The fourth paragraph compares the proposed study with existing literature using citations from the selected references. The comparison should be specific: what prior work achieved, what data or validation it lacked, and how this study extends or differs from it.

The fifth paragraph states significance, limitations, and future work. It should return to the Introduction gaps and explain what remains unresolved, such as cross-region generalization, access to field labels, disease-specific validation, and the need for prospective field trials.

References and Citation Output

Markdown output should use author-year citations in the body text. LaTeX output should be paired with a generated `.bib` file derived from Zotero or supplemental metadata. The `.tex` file should use BibTeX-compatible citation management so the draft can be moved into an Overleaf journal template with minimal manual work.
